API Reference

Public endpoints & authenticated API

Base URL

All responses are JSON. success: true/false is always present.


Base: https://kinder.astro.ncu.edu.tw

Authentication

  • Public API — No key needed. Rate limited to 1 req/s per IP per endpoint.
  • Authenticated API — Pass API key via X-API-Key header or ?api_key= query param.

Public API

No key required  ·  1 req/s per IP
GET /api/distance Luminosity distance & absolute magnitude

Calculate luminosity distance from redshift using a flat ΛCDM cosmology. Optionally compute absolute magnitude when apparent magnitude is provided. Cosmological parameters default to Planck 2018.

ParameterDescriptionDefault
zrequiredRedshift
z_erroptionalRedshift uncertainty → adds distance error to response
moptionalApparent magnitude → enables absolute magnitude output
AoptionalExtinction (magnitudes)0
H0optionalHubble constant (km/s/Mpc)67.7
Om0optionalMatter density Ωm0.309
Tcmb0optionalCMB temperature (K)2.725
Cosmology reference: Planck Collaboration 2020, A&A, 641, A6 — ADS
Example request
/api/distance?z=0.039&m=15.3&A=0.1
Example response
{ "success": true, "result": { "input": { "z": 0.039, "z_err": null, "m": 15.3, "A": 0.1 }, "cosmology": { "H0": 67.7, "Om0": 0.309, "Tcmb0": 2.725 }, "distance": { "distance_mpc": 171.23, "distance_gpc": 0.1712, "distance_pc": 171230000.0, "distance_ly": 558511800.0, "distance_km": 5.28e21, "redshift": 0.039 }, "magnitude": { "absolute_magnitude": -18.52, "distance_modulus": 36.17, "k_correction": 0.17, "distance_mpc": 171.23, "extinction": 0.1 } } }
GET /api/coords RA / DEC coordinate conversion

Convert between HMS/DMS sexagesimal and decimal degree formats. RA and DEC can be converted in the same request.

ParameterDescription
ra_hmsoptionalRA in hh:mm:ss.s → returns decimal degrees
ra_degoptionalRA in decimal ° → returns HMS
dec_dmsoptionalDEC in ±dd:mm:ss → returns decimal degrees
dec_degoptionalDEC in decimal ° → returns DMS
Provide ra_hms or ra_deg, and/or dec_dms or dec_deg. At least one parameter is required.
Example request
/api/coords?ra_hms=12:34:56.78&dec_dms=-23:45:12.34
Example response
{ "success": true, "result": { "ra_hms": "12:34:56.78", "ra_decimal": 188.73658, "dec_dms": "-23:45:12.34", "dec_decimal": -23.75343 } }
GET /api/date MJD / JD / calendar date conversion

Convert between Modified Julian Date (MJD), Julian Date (JD), and calendar date (UTC). Provide exactly one of the three parameters.

ParameterDescription
mjdoptionalModified Julian Date (days since 1858-11-17)
jdoptionalJulian Date
dateoptionalISO 8601 calendar date, e.g. 2024-06-01T12:00:00
Example request
/api/date?mjd=59000.5
Example response
{ "success": true, "result": { "mjd": 59000.5, "jd": 2459001.0, "common_date": "2020-08-02 12:00:00 UTC" } }
GET /api/finding_chart/image Finding chart — PNG image direct download

Returns a finding chart PNG directly (Content-Type: image/png). Pulls imagery from DSS / DESI LS / Pan-STARRS and overlays Tycho-2 + UCAC4 star annotations. Rate limit: 1 request per 30 seconds per IP (network-heavy).

ParameterDescriptionDefault
obj_nameoptional*Look up RA/Dec from database — accepts full name (AT2025wny) or year+letters (2025wny)
raoptional*Right Ascension — HMS (hh:mm:ss.s) or decimal degrees
decoptional*Declination — DMS (±dd:mm:ss) or decimal degrees
nameoptionalCustom display label on chart — defaults to DB full name when obj_name is usedTarget
surveyoptionalDSS2 Red · DSS2 Blue · DSS2 IR · DESI-color · DESI-r · PS1-color · PS1-r …DESI-color
fovoptionalField of view in arcmin10
invertoptional1 = white background (single-band surveys only)0
Star Annotation Controls
mag_limitoptionalFaintest star to annotate (magnitude)15
show_magoptional1 = show magnitude labels, 0 = hide them1
show_namesoptional1 = show bright star names, 0 = hide them1
* Provide obj_name or both ra + dec.  |  See all available surveys: GET /api/finding_chart/surveys
URL (basic with all star annotations)
/api/finding_chart/image?ra=12:34:56.78&dec=-23:45:12.34&survey=DESI-color&fov=10&name=SN2025abc
URL (no star labels)
/api/finding_chart/image?ra=12:34:56.78&dec=-23:45:12.34&name=SN2025abc&show_mag=0&show_names=0
wget
wget -O SN2025abc_chart.png \ "https://kinder.astro.ncu.edu.tw/api/finding_chart/image?ra=12:34:56.78&dec=-23:45:12.34&name=SN2025abc&survey=DESI-color&fov=10&show_mag=1&show_names=1"
curl
curl -o SN2025abc_chart.png \ "https://kinder.astro.ncu.edu.tw/api/finding_chart/image?ra=12:34:56.78&dec=-23:45:12.34&name=SN2025abc&survey=DESI-color&show_mag=1&show_names=1&mag_limit=13"
Python
import requests r = requests.get("https://kinder.astro.ncu.edu.tw/api/finding_chart/image", params={ "ra": "12:34:56.78", "dec": "-23:45:12.34", "name": "SN2025abc", "survey": "DESI-color", # Image source "fov": 10, # Arcminutes "show_mag": 1, # Show magnitudes "show_names": 1, # Show star names "mag_limit": 15, # Only annotate stars brighter than this }) open("chart.png", "wb").write(r.content)
Star annotation: Use show_mag=0 to hide magnitudes, show_names=0 to hide star names, or mag_limit=12 to only show brighter stars.  |  Invert colors: invert=1 for white background (single-band surveys only)
GET /api/visibility/image Nightly visibility plot — JPEG image direct download

Returns an altitude-vs-time observing track plot as JPEG (Content-Type: image/jpeg). Defaults to Lulin Observatory (NCU, Taiwan, UTC+8). Pass coordinates for any observing location: lon (longitude), lat (latitude), alt (altitude), tz (timezone). Rate limit: 1 req / 15 s per IP.

ParameterDescriptionDefault (Lulin)
daterequiredObservation date YYYY-MM-DD
obj_nameoptional*Look up RA/Dec from database — accepts full name (AT2025wny) or year+letters (2025wny)
raoptional*Target RA — HMS (hh:mm:ss) or decimal °
decoptional*Target Dec — DMS (±dd:mm:ss) or decimal °
nameoptionalCustom display label on plot — defaults to DB full name when obj_name is usedTarget
Observatory Location — Choose a preset or specify custom coordinates
lonoptionalLongitude — DMS (±dd:mm:ss) or decimal °. East positive, West negative.120:52:21.5
latoptionalLatitude — DMS (±dd:mm:ss) or decimal °. North positive, South negative.23:28:10.0
altoptionalAltitude in metres above sea level2800
tzoptionalUTC offset (integer, −12 to +14). Negative for West, positive for East.8
* Provide obj_name or both ra + dec.
Observatory location presets (copy-paste the coordinates):
Lulin (NCU, Taiwan) lon=120:52:21.5 lat=23:28:10.0 alt=2800 tz=8
Las Campanas, Chile lon=-70:41:33.0 lat=-29:00:51.12 alt=2282 tz=-3
Palomar, California lon=-116:51:50.0 lat=33:21:21.0 alt=1706 tz=-8
URL (Lulin Observatory, default location)
/api/visibility/image?date=2026-06-07&ra=12:34:56.78&dec=-23:45:12.34&name=SN2025abc
URL (Las Campanas, Spain, custom location)

URL (Custom location: Tokyo, Japan)
/api/visibility/image?date=2026-06-07&ra=12:34:56.78&dec=-23:45:12.34&name=SN2025abc&lon=139:46:00&lat=35:40:00&alt=0&tz=9
wget (custom location)
wget -O visibility.jpg \ "https://kinder.astro.ncu.edu.tw/api/visibility/image?date=2026-06-07&ra=12:34:56.78&dec=-23:45:12.34&name=SN2025abc&lon=-70:41:33&lat=-29:00:51&alt=2282&tz=-3"
Python (with custom location)
import requests r = requests.get("https://kinder.astro.ncu.edu.tw/api/visibility/image", params={ "date": "2026-06-07", "ra": "12:34:56.78", "dec": "-23:45:12.34", "name": "SN2025abc", # Custom location: Las Campanas "lon": "-70:41:33", # West (negative) "lat": "-29:00:51", # South (negative) "alt": 2282, # metres "tz": -3, # UTC-3 }) open("visibility.jpg", "wb").write(r.content)
Location format: Use DMS (±dd:mm:ss) or decimal degrees both work. East/North are positive, West/South are negative.  |  No presets? Find your location's lon/lat/tz and use the custom parameters.
GET /api/objects/{name} Object metadata (+ photometry & spectra with key)

Returns object metadata (coordinates, redshift, type, discovery info, etc.) for anyone. Pass a valid api_key to also receive photometry and spectroscopy, filtered to only the sources your account has permission to access.

ParameterDescription
{name}pathObject name — full (AT2025wny, SN2024abc) or year+letters (2025wny)
api_keyoptionalYour API key. When valid, adds photometry and spectra arrays to the response
Without key — returns metadata only (public fields).
With key — also returns photometry[] and spectra[], each filtered to sources you have access to.
Public (no key)
/api/objects/2025wny
With API key
/api/objects/2025wny?api_key=YOUR_KEY
Python
import requests # Public (metadata only) r = requests.get("https://your-domain.com/api/objects/2025wny") # Authenticated (+ photometry + spectra) r = requests.get("https://your-domain.com/api/objects/2025wny", params={"api_key": "YOUR_KEY"}) print(r.json())
Response (no key)
{ "success": true, "name": "AT2025wny", "metadata": { "name": "AT2025wny", "name_prefix": "AT", "type": "AT", "ra_deg": 188.7366, "dec_deg": -23.7534, "ra_hms": "12:34:56.78", "dec_dms": "-23:45:12.3", "discovery_date": "2025-04-15", "discovery_mag": 18.5, "reporting_group": "ZTF", "redshift": 0.039, "distance_mpc": 171.23, "brightest_abs_mag": -18.52, "internal_names": "ZTF25abc", "status": "followup" } }
Response (with key, extra fields)
{ "success": true, "name": "AT2025wny", "metadata": { ... }, "photometry": [ { "mjd": 60750.5, "magnitude": 18.32, "magnitude_error": 0.04, "filter": "rp", "telescope": "SLT", "is_upper_limit": false }, ... ], "photometry_count": 42, "spectra": [ { "source": "LOT/SPRAT", "mjd": 60752.0, "label": "LOT/SPRAT @ MJD 60752.0" } ], "spectra_count": 1, "requested_by": "user@example.com" }

Authenticated API

API Key required

How to authenticate

  • Via HTTP header: X-API-Key: your_key_here
  • Via query param: ?api_key=your_key_here
  • Generate your key from the account settings page (authorized users only).
GET /api/test Validate API key

Test whether an API key is valid. Returns the associated user information on success.

Example request
curl -H "X-API-Key: YOUR_KEY" https://kinder.astro.ncu.edu.tw/api/test
Example response
{ "success": true, "message": "API Authentication successful!", "user": { "name": "Alex Lee", "email": "user@example.com", "role": "admin", "is_admin": true, "groups": ["GREAT Lab"] } }

Retrieve the current active observation target list for SLT, LOT, or both telescopes.

ParameterDescription

POST /api/v1/observation_logs Add / update / delete a log entry

Upsert (insert or update) an observation log entry. Set "action":"delete" to delete instead.

Field (JSON body)Description
target_namerequiredObject name, e.g. SN2025abc
obs_daterequiredObservation date YYYY-MM-DD
actionoptionalupsert (default) or delete
telescopeoptionalSLT or LOT
is_triggeredoptionalWas observation triggered? (bool)
trigger_filteroptionalFilter used for trigger, e.g. rp
trigger_expoptionalTrigger exposure time (seconds)
trigger_countoptionalNumber of trigger frames
is_observedoptionalWas the object observed? (bool)
observed_filteroptionalFilter actually used
observed_expoptionalActual exposure time (seconds)
observed_countoptionalNumber of frames taken
user_nameoptionalObserver name (auto-filled from API key owner)
priorityoptionalNormal / High / Urgent / Filler
Upsert example
curl -X POST \ -H "X-API-Key: YOUR_KEY" \ -H "Content-Type: application/json" \ -d '{ "target_name": "SN2025abc", "obs_date": "2026-06-07", "telescope": "SLT", "is_triggered": true, "trigger_filter": "rp", "trigger_exp": 300, "trigger_count": 12, "is_observed": true }' \ https://kinder.astro.ncu.edu.tw/api/v1/observation_logs
Delete example
curl -X POST \ -H "X-API-Key: YOUR_KEY" \ -H "Content-Type: application/json" \ -d '{ "action": "delete", "target_name": "SN2025abc", "obs_date": "2026-06-07" }' \ https://kinder.astro.ncu.edu.tw/api/v1/observation_logs
